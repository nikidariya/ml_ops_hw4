# MLOps Homework 4

*Автор: Никитина Дарья Дмитриевна*

## Что сделано
- Обучена модель ResNet18 на датасете AI vs Human Generated Images (Train_1)
- Интеграция с TensorBoard: логируются loss, accuracy, F1, precision, recall
- Развёрнуто локальное S3 хранилище MinIO через Docker
- Модели сохраняются и загружаются из MinIO
- Построен DVC пайплайн для дообучения и тестирования

## Почему часть работы выполняется локально
Обучение базовой модели проводилось в Kaggle на GPU, так как локальный
ноутбук не имеет видеокарты. MinIO, DVC пайплайн и дообучение
разворачиваются локально, так как это инструменты для управления
экспериментами, а не для вычислений - им GPU не нужен.

## Где что искать
- `train_model.ipynb` - основной ноутбук с обучением, оценкой и выводами
- `scripts/finetune.py` - дообучение модели на Train_2
- `scripts/test_model.py` - тестирование дообученной модели на Test_2
- `dvc.yaml` - описание DVC пайплайна
- `dvc.lock` - зафиксированные версии пайплайна
- `docker-compose.yml` - запуск MinIO локально
- `upload_model.py` - загрузка модели в MinIO
- `metrics_v2.json` - метрики дообученной модели

## Как воспроизвести
1. `docker compose up -d` - запустить MinIO
2. `dvc repro` - запустить пайплайн дообучения
3. `tensorboard --logdir runs/` - посмотреть графики
